### 1.2 建议优化项

#### 优化 1：为平板端提供中间模式
**问题：** 当前只有桌面端（全功能）和移动端（精简）两种模式，平板端（iPad）屏幕较大，可以支持更多功能。

**建议：**
```typescript
// src/utils/platform.ts
export function isTablet(): boolean {
    // 检测平板设备（屏幕宽度 > 768px 的移动设备）
    return isMobile() && window.innerWidth >= 768;
}

export function getPlatformTier(): 'desktop' | 'tablet' | 'mobile' {
    if (isDesktop()) return 'desktop';
    if (isTablet()) return 'tablet';
    return 'mobile';
}
```

**平板端可启用的功能：**
- ✅ 伏笔面板（屏幕够大）
- ✅ 时间线面板（屏幕够大）
- ✅ 状态视图面板（屏幕够大）
- ❌ 悬浮便签（触摸拖拽体验不佳）
- ❌ OBS 服务器（无实际需求）
- ❌ Worker 时间追踪（省电考虑）

#### 优化 2：移动端字数统计优化
**问题：** 移动端没有文件夹字数缓存，但用户可能需要查看文件夹总字数。

**建议：**
- 保留文件夹字数显示功能，但使用轻量级实时计算
- 只在用户点击文件夹时计算，不预先缓存
- 添加加载指示器

#### 优化 3：移动端右键菜单增强
**问题：** 移动端只有"设定目标字数"一个菜单项，功能过于单一。

**建议添加：**
- 查看当前文件字数统计
- 查看今日写作统计
- 快速设置当日目标

#### 优化 4：触摸优化
**问题：** 当前使用固定的 44px 触摸目标，但某些按钮可能更小。

**建议：**
- 审查所有按钮和可点击元素
- 确保最小触摸目标 44px（iOS HIG 标准）
- 增加按钮间距避免误触

### 2.3 性能优化机会

#### 优化 1：防抖策略优化
**当前状态：**
```typescript
// editor-change: 300ms
// folder-refresh: 500ms
// word-count-update: 100ms
```

**建议：** 根据用户输入速度动态调整防抖时间
```typescript
class AdaptiveDebounce {
    private lastInputTime = 0;
    private inputSpeed = 0;
    
    getOptimalDelay(): number {
        const now = Date.now();
        const delta = now - this.lastInputTime;
        this.lastInputTime = now;
        
        // 快速输入时延长防抖时间
        if (delta < 100) return 500;
        if (delta < 300) return 300;
        return 100;
    }
}
```

#### 优化 2：缓存持久化优化
**问题：** 每次修改文件都会触发缓存保存，可能导致频繁磁盘 I/O。

**建议：**
- 使用更长的防抖时间（5 秒）
- 批量保存缓存更新
- 只在插件卸载时强制保存

#### 优化 3：时间线和伏笔面板性能
**问题：** 每次文件修改都会刷新面板，可能导致不必要的重新渲染。

**建议：**
- 只在相关文件修改时刷新
- 使用虚拟滚动处理大量条目
- 缓存渲染结果
