## 快速开始

### 安装方法

#### 方法 1: 使用 BRAT（推荐）
1. 安装 [BRAT](https://github.com/TfTHacker/obsidian42-brat) 插件
2. 在 BRAT 设置中添加此仓库：`HatanoChihiro/obsidian-webnovel-assistant`
3. 启用插件

#### 方法 2: 手动安装
1. 从 [Releases](https://github.com/HatanoChihiro/obsidian-webnovel-assistant/releases) 下载最新版本的 `main.js` 和 `manifest.json`
2. 在 Obsidian 库中创建文件夹：`.obsidian/plugins/web-novel-assistant/`
3. 将下载的文件放入该文件夹
4. 重启 Obsidian，在设置 → 第三方插件中启用

### 首次使用

1. 打开任意 Markdown 文件
2. 状态栏会显示当前文件的字数统计
3. 点击状态栏可以设置目标字数

## 核心功能

### 工作区设置

#### 功能说明
- **自定义插件工作区**：指定插件工作的文件夹范围
- **影响范围**：
  - ✅ 字数统计（状态栏、文件浏览器）
  - ✅ 写作状态面板的历史记录
  - ✅ 文件修改监听和缓存构建
  - ❌ 不影响：伏笔、时间线、便签等功能

#### 使用方法
1. 打开插件设置
2. 找到 **工作区文件夹**
3. 输入文件夹路径（如 `小说/第一部`）
4. 多个文件夹用逗号分隔（如 `小说/第一部, 小说/第二部`）
5. 留空则全局生效（所有文件都统计）

#### 使用场景
- 同一个库中有多个项目，只想统计特定项目的字数
- 避免笔记、日记等非小说内容影响统计
- 提高性能（只监听和缓存指定文件夹）

### 字数统计

#### 功能说明
- **精确统计中文字数**：准确识别中文、英文、数字
- **实时更新**：编辑时自动更新字数
- **净增字数**：显示本次会话新增的字数
- **状态栏显示**：在 Obsidian 底部状态栏显示

#### 使用方法
1. 打开任意 Markdown 文件
2. 状态栏自动显示字数：`📝 1,234 字 | 净增: +56 字`
3. 点击状态栏可以快速设置目标

#### 相关设置
- **工作区文件夹**：限制统计范围（详见 [工作区设置](#工作区设置)）

#### 统计规则
- ✅ 统计：中文字符、英文单词、数字
- ❌ 不统计：Markdown 语法、代码块、注释、YAML frontmatter

### 目标追踪

#### 功能说明
- **章节目标**：为每个章节设置独立的目标字数
- **每日目标**：设置每日写作目标
- **进度显示**：实时显示完成百分比
- **进度条**：可视化显示进度

#### 使用方法

##### 设置章节目标
1. 右键点击文件 → **设定本章目标字数**
2. 输入目标字数（如 3000）
3. 状态栏显示进度：`📝 1,234 / 3,000 (41%)`

##### 设置每日目标
1. 打开插件设置
2. 找到 **每日目标字数**
3. 输入目标（如 5000）

#### 相关设置
- **默认目标字数**：新文件的默认目标（默认 3000 字）
- **每日目标字数**：每日写作目标（默认 5000 字）
- **显示目标进度**：是否在状态栏显示进度（桌面端专属，默认启用）

##### 查看进度
- **状态栏**：显示当前文件进度
- **写作状态面板**：显示详细统计（点击左侧 Ribbon 图标）

### 时间追踪

#### 功能说明
- **专注时间**：统计实际写作时间
- **摸鱼时间**：统计空闲时间
- **自动检测**：60 秒无操作自动切换为摸鱼
- **历史记录**：保存每日统计数据

#### 使用方法

##### 开始追踪
1. 打开命令面板（Ctrl/Cmd + P）
2. 输入 `开始/暂停 摸鱼时间统计`
3. 开始写作，插件自动追踪时间

##### 查看统计
- **状态栏**：显示今日专注时间
- **写作状态面板**：显示详细时间分布
- **历史统计**：点击面板中的 "查看历史统计"

##### 重置统计
- 命令面板 → `重置直播统计数据`
- 清空当前会话的时间和字数统计

#### 相关设置
- **空闲超时阈值**：多少毫秒无操作算作摸鱼（默认 60000 = 60 秒）

> **注意**：移动端不支持时间追踪功能（需要 Worker 支持）

### 文件浏览器字数显示

#### 功能说明
- **文件夹字数**：显示文件夹内所有文件的总字数
- **文件字数**：显示单个文件的字数
- **实时更新**：修改文件后自动更新
- **工作区过滤**：只显示指定文件夹的字数
- **性能优化**：使用缓存机制，避免频繁计算

> **注意**：此功能默认关闭，因为在大型项目中可能影响性能。如需使用，请在设置中手动启用。

#### 使用方法

##### 启用功能
1. 打开插件设置
2. 启用 **文件浏览器字数统计**
3. 插件会自动构建缓存（可能需要几秒钟）
4. 文件浏览器中会显示字数：`第一章 (3,456)`

##### 重建缓存
如果字数显示不正确：
1. 命令面板 → `重建文件夹字数缓存`
2. 等待重建完成

#### 相关设置
- **文件浏览器字数统计**：是否启用此功能（默认关闭，避免性能问题）
- **工作区文件夹**：限制统计范围（详见 [工作区设置](#工作区设置)）

### 智能章节排序

#### 功能说明
- **自动排序**：按章节编号自动排序
- **多种格式**：默认支持阿拉伯数字、中文数字
- **自定义规则**：支持自定义正则表达式
- **实时生效**：创建/重命名文件后自动排序
- **兼容性**：与 manual-sorting 插件兼容，建议配合使用

> **💡 推荐配置**：
> 建议同时安装 [manual-sorting](https://github.com/Silejonu/obsidian-custom-sort) 插件，两者配合使用效果更佳：
> - **manual-sorting**：手动调整非章节文件的顺序（如前言、后记、设定等）
> - **智能章节排序**：自动排序章节文件（如第1章、第2章等）
> - 两者互不冲突，可以同时启用

#### 使用方法

##### 启用功能
1. 打开插件设置
2. 启用 **智能章节排序**
3. 文件浏览器中的章节会自动排序

##### 支持的格式
- 阿拉伯数字：`第1章`、`第01章`、`Chapter 1`
- 中文数字：`第一章`、`第二十三章`、`第一百章`、`第九百九十九章`
- 纯数字：`1`、`01`、`001`
- 自定义规则

##### 自定义规则
1. 打开插件设置
2. 找到 **章节命名规则**
3. 点击 **添加规则**
4. 输入规则名称和正则表达式
5. 启用/禁用规则
6. 拖动调整规则优先级

##### 默认预设规则

插件已经预设了 3 个常用规则，可以直接使用：

**规则 1：阿拉伯数字（第1章、第01章）**
```
名称：阿拉伯数字（第1章、第01章）
正则：^第?(\d+)[章节回卷部册篇]?
启用：是
```
支持格式：`第1章`、`第01章`、`1章`、`第1节`、`第1回`、`第1卷`、`第1部`、`第1册`、`第1篇`

**规则 2：中文数字（第一章、第二章）**
```
名称：中文数字（第一章、第二章）
正则：^第?([零一二三四五六七八九十百千万壹贰叁肆伍陆柒捌玖拾佰仟萬〇]+)[章节回卷部册篇]?
启用：是
```
支持格式：`第一章`、`第二十三章`、`第一百章`、`第九百九十九章`（支持一到九百九十九）

**规则 3：纯数字（1、01、001）**
```
名称：纯数字（1、01、001）
正则：^(\d+)$
启用：是
```
支持格式：`1`、`01`、`001`（纯数字文件名，无前缀后缀）

##### 自定义规则示例

除了默认规则，你还可以添加自己的规则：

**小数点章节**（如：1.1、1.2、2.1、2.2）
```
名称：小数点章节
正则：^(\d+\.\d+)
启用：是
```

**英文章节**（如：Chapter 1、Chapter 01）
```
名称：英文章节
正则：^[Cc]hapter\s*(\d+)
启用：是
```

**卷章节**（如：第一卷第一章）
```
名称：卷章节
正则：^第.+卷第?(\d+)[章节]?
启用：是
```

> **提示**：
> - 正则表达式中的 `(\d+)` 或 `(\d+\.\d+)` 用于捕获章节编号
> - 规则的顺序决定了排序的优先级（上面的规则优先）
> - 可以同时启用多个规则，插件会按顺序匹配

> **⚠️ 重要提醒**：
> 如果同时启用了**数字规则**（如 `^(\d+)$`）和**小数点规则**（如 `^(\d+\.\d+)`），必须将小数点规则放在数字规则**上方**，否则小数点章节会被数字规则优先匹配，导致排序错误。
> 
> 例如：`1.1` 会被 `^(\d+)$` 匹配为 `1`，而不是被 `^(\d+\.\d+)` 匹配为 `1.1`。
> 
> **正确顺序**：
> 1. 小数点章节 `^(\d+\.\d+)` ✅
> 2. 纯数字 `^(\d+)$` ✅

##### 自动创建下一章
1. 打开当前章节文件
2. 命令面板 → `自动创建下一章`
3. 自动创建下一个章节文件

##### 合并章节
1. 右键点击文件夹 → `合并章节`
2. 插件会自动扫描文件夹内的所有章节文件
3. 按章节编号排序后合并到一个新文件
4. 自动打开合并后的文件

> **注意**：
> - 只会合并**能被章节排序规则识别**的文件
> - 例如：`第1章.md`、`第一章.md` 会被合并
> - 但 `前言.md`、`后记.md`、`设定.md` 等不会被合并
> - 这样可以避免误合并非章节内容
> - 如果需要合并特定文件，请为其添加匹配的章节命名规则

#### 相关设置
- **启用智能章节排序**：是否启用自动排序（默认关闭）
- **章节命名规则**：自定义章节识别规则（支持正则表达式）

## 高级功能

### 悬浮便签

#### 功能说明
- **浮动窗口**：可拖动、可调整大小的便签
- **多主题**：6 种预设主题颜色
- **自动保存**：关闭时自动保存位置和内容
- **关联文件**：可以关联到具体文件

#### 使用方法

##### 创建便签
**方法 1：空白便签**
- 点击左侧 Ribbon 图标（便签图标）
- 或命令面板 → `新建空白悬浮便签`

**方法 2：从文件创建**
- 右键点击文件 → `抽出为便签`

**方法 3：从选中文字创建**
- 选中文字 → 右键 → `抽出为便签`

##### 编辑便签
- **编辑模式**：点击铅笔图标
- **预览模式**：点击眼睛图标
- **更换主题**：点击调色板图标
- **固定位置**：点击图钉图标

##### 保存便签
- **保存到文件**：点击保存图标
- **关联文件**：如果便签来自文件，保存会更新原文件
- **新建文件**：如果是空白便签，会提示创建新文件

##### 关闭便签
- 点击关闭按钮（X）
- 如果有未保存的内容，会提示保存

> **注意**：
> - 便签的本意是方便作为创作参考和记录部分当前章节的碎片化线索，存在一定的阅后即焚作用，因此不会自动保存
> - 如果窗口中存在未保存的便签而你需要保存，请不要直接关闭Obsidian，内容会丢失

#### 相关设置
- **便签主题**：6 种预设主题颜色，可自定义
- **便签透明度**：背景透明度（0-1，默认 0.9）

### 伏笔管理

#### 功能说明
- **标注伏笔**：标记需要回收的伏笔
- **多章节回收**：一个伏笔可以在多个章节回收
- **标签分类**：使用标签组织伏笔
- **时间戳**：记录创建和回收时间

#### 使用方法

##### 标注伏笔
1. 选中需要标注的文字
2. 右键 → `标注为伏笔`
3. 或使用快捷键（需在设置中自定义）
4. 输入伏笔描述和标签
5. 点击保存

##### 查看伏笔
1. 点击左侧 Ribbon 图标（书签图标）
2. 或命令面板 → `打开/关闭伏笔面板`
3. 面板显示所有伏笔

##### 回收伏笔
**方法 1：在伏笔面板中**
1. 找到要回收的伏笔
2. 点击 **标记已回收**
3. 选择回收章节（可多选）
4. 点击确定

**方法 2：在伏笔文件中**
1. 打开伏笔文件（默认名称：`伏笔.md`）
2. 将光标放在伏笔条目上
3. 命令面板 → `标记伏笔已回收`
4. 选择回收章节

#### 相关设置
- **伏笔文件名**：默认 `伏笔`，可自定义
- **显示时间戳**：是否显示创建时间（默认启用）
- **默认标签**：预设常用标签（默认：人物、情节、世界观、道具、线索）

**章节**: [[第五章]]

## 移动端功能

### 复制全文

#### 功能说明
- **解决 Obsidian 移动端限制**：Obsidian 移动端的全选功能只能选择可视范围内的文本
- **一键复制全文**：无论文档多长，都能完整复制
- **快速访问**：通过命令面板或文件菜单调用
- **移动端专属**：桌面端使用 Ctrl/Cmd + A 全选更方便

#### 使用方法

##### 方法 1：命令面板
1. 打开命令面板（通常是屏幕左上角的命令图标）
2. 输入 `复制全文`
3. 点击执行
4. 显示提示：`✅ 已复制全文（xxx 字符）`

##### 方法 2：文件菜单
1. 在文件列表中长按文件
2. 选择 `复制全文`
3. 完整内容已复制到剪贴板

> **注意**：此功能仅在移动端可用。桌面端请使用 Ctrl/Cmd + A 全选后复制。

#### 适用场景
- 📱 需要将整篇文章复制到其他应用
- 📱 文档很长，手动全选不方便
- 📱 需要快速备份文本内容
- 📱 分享文章到社交媒体或聊天工具

### 浮动字数统计

#### 功能说明
- **移动端专属**：在屏幕上显示浮动的字数统计窗口
- **实时更新**：编辑时自动更新字数
- **显示进度**：包含当前字数、章节目标、进度百分比
- **不遮挡编辑**：窗口位置可调整

#### 使用方法
1. 打开插件设置
2. 找到 **显示移动端浮动字数统计**
3. 开启后，浮动窗口自动显示在屏幕上
4. 包含：当前字数、目标字数、完成百分比

> **提示**：启用浮动窗口后，状态栏的字数统计会自动隐藏，避免重复显示。

#### 使用方法

3. URL 输入：`http://127.0.0.1:24816/`

5. 勾选 **刷新浏览器当场景变为活动状态**

## 常见问题

### Q: 字数统计不准确？
A: 
1. 检查是否启用了 **文件浏览器字数统计**
2. 尝试 **重建文件夹字数缓存**
3. 确认文件编码为 UTF-8
4. 字数统计采用中文小说网站的字符计算法则，可能与 Word、WPS 等其他编辑软件计算不相符，可另外搭配 Obsidian 自带的字数统计插件使用

### Q: 章节排序不生效？
A:
1. 检查是否启用了 **智能章节排序**
2. 确认文件名符合规则格式
3. 尝试 **手动刷新章节排序**

### Q: OBS 叠加层无法显示？
A:
1. 检查是否启用了 **OBS 叠加层**
2. 确认端口没有被占用
3. 在浏览器中访问 `http://127.0.0.1:24816/` 测试
4. 检查防火墙设置

### Q: 伏笔文件找不到？
A:
1. 伏笔文件在当前文件的同级目录
2. 文件名默认为 `伏笔.md`
3. 可以在设置中修改文件名

### Q: 时间追踪不准确？
A:
1. 确认已开启时间追踪（命令面板）
2. 检查 **空闲超时阈值** 设置
3. 60 秒无操作会自动切换为摸鱼时间

### Q: 便签内容丢失？
A:
1. 关闭便签时会提示保存
2. 便签状态保存在插件设置中
3. 建议及时保存到文件

### Q: 移动端功能受限？
A:
- 移动端只提供基础功能（字数统计、目标追踪、复制全文）
- 高级功能（Worker 时间追踪、OBS、便签）仅在桌面端可用
- 平板端支持面板功能（写作状态、伏笔、时间线）
- 移动端提供"复制全文"命令，解决 Obsidian 移动端全选只能选择可视范围的问题

### Q: 与其他插件冲突？
A:
- 智能章节排序与 **manual-sorting** 插件完全兼容
- **推荐配合使用**：
  - manual-sorting 负责手动排序非章节文件（前言、后记等）
  - 智能章节排序负责自动排序章节文件（第1章、第2章等）
- 两者互不冲突，可以同时启用
- 如有其他插件冲突，请在 GitHub 提交 Issue
